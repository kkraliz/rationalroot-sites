Claude Desktop Architecture

How Chat, Cowork, and Code relate to each other — and where context crosses (or doesn't cross) the boundaries between them.

Eyes Only
Diagram 1

The Contextual Cascade

Each mode operates in its own context silo. Context flows downward through explicit bridges — never automatically, never backwards.

Chat

claude.ai & Desktop App
  • Conversational AI with Projects
  • User-level memory across all chats
  • Project Knowledge (files, instructions)
  • No file system or shell access
  • Artifacts (code, documents, diagrams)
  • Web search, tool use via MCP
Own memory system
Bridge

Project Import

Attach a Chat project to a Cowork space to bring its knowledge files and instructions into Cowork's context. This is a read-only, one-way link — Cowork can see the knowledge but Chat doesn't receive Cowork's memory back.

Chat → Cowork ▶

Cowork

Desktop App — Agentic Mode
  • File-based memory per space
  • Mounted folder access (read/write)
  • Sandboxed Linux shell
  • Skills, plugins, MCP tools
  • CLAUDE.md instructions apply
  • Can import Chat project knowledge
Own memory system
Shared Layer

CLAUDE.md

Both Cowork and Code read CLAUDE.md files for persistent instructions. If a Cowork space is pointed at the same directory as a Code project, they share those instructions — but their runtime memories remain separate.

Cowork ↔ Code ↔

Code

CLI Terminal Tool
  • CLAUDE.md as primary memory/config
  • Full file system access
  • Shell access (direct, not sandboxed)
  • Git-aware context
  • MCP servers, hooks, slash commands
  • No centralized "memory" feature
CLAUDE.md-driven
Chat ↔ Code: No direct connection. Chat doesn't read CLAUDE.md and Code doesn't access Chat's memory. They share only an Anthropic account.

🔗 How CLAUDE.md Applies Across Environments

CLAUDE.md is a shared instruction layer — but it doesn't work everywhere equally.

Chat
Not supported
Chat uses Project Instructions and user-level memory instead. It doesn't read CLAUDE.md files from disk.
Cowork
Fully supported
Reads CLAUDE.md from the mounted workspace folder. Also has its own file-based memory system alongside it.
Code
Primary mechanism
CLAUDE.md is the core way to persist instructions. Supports project, user, and enterprise levels.

Key Takeaway

There is no single unified memory across Chat, Cowork, and Code. Each operates in its own context silo. The main bridge is importing a Chat project into Cowork — this brings Chat's knowledge into Cowork's context as read-only data. CLAUDE.md is the closest thing to a shared layer, but it only works in Cowork and Code (not Chat). If you want continuity across modes, the project import is the mechanism to use.
